# LLM Usage Report

## Overview

AI/LLM tools were used throughout Week 2 for several tasks related to meeting processing, documentation, and content formatting. This report summarizes all LLM-assisted activities.

---

## Usage Breakdown

### 1. Meeting Transcription

**Tool:** LLM-based transcription service

The raw video recording of the customer meeting was processed through a neural network to generate an initial transcript. The output was a set of raw, unformatted speech segments with timestamps that required further processing.

### 2. Transcript Processing & Editing

**Tool:** LLM-based assistant

The raw transcript was processed to:
- Merge fragmented speech segments into coherent, meaningful phrases.
- Interpret and resolve ambiguous or unclear moments in the conversation.
- Preserve logical flow and readability of the dialogue.
- Assign speaker identities to each segment (Serafim Soldatov, Nikita Maksimenko, Maxim Maltsev, Lyubov Savchenko, Maxim Merkushev).

### 3. Proofreading & Accuracy Review

**Tool:** LLM-based assistant

The processed transcript and all derived documentation (meeting summary, weekly analysis, MVP v0 report) were reviewed using an LLM to:
- Detect and correct typographical errors.
- Identify factual inaccuracies or inconsistencies between documents.
- Ensure terminology consistency across all reports.

### 4. Translation

**Tool:** LLM-based assistant

The full meeting transcript was translated from Russian to English, preserving all timecodes, speaker attributions, and formatting.

### 5. Format Conversion

**Tool:** LLM-based assistant

Various inputs in different formats — raw speech segments, structured notes, API endpoint lists, and README files — were converted into a unified Markdown (`.md`) format for consistent documentation across the project repository.
